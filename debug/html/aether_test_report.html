BUG: publish_collab_delta kräver url i nested objects
agent-reader misslyckades med "missing field 'url' at line 1 column 610". API-valideringen kräver url-fältet i nested delta-objekt inuti store_json, vilket inte är dokumenterat. Kan fixas med bättre API-dokumentation eller mer förlåtande validering.
⚠️
fetch_extract missar priser utan ARIA-labels
Tidigare test visade att extract-funktionen missar priser i <p class="price_color"> utan ARIA-label. Goal-relevance scoring matchar felaktigt på warning-banners. Lösning: regex-fallback på £/$/€-mönster i text-noder + role-blacklist för generic warning-containers.
🐛 BUG #1: publish_collab_delta – nested url validation
Reproduceras: Skicka store_json där ett befintligt delta-objekt saknar url-fältet.
Felmeddelande: Invalid collab store: missing field 'url' at line 1 column 610
Root cause: Servern deserialiserar hela store_json inkl. befintliga deltan och validerar alla nested delta-objekt. Om ett nested delta saknar url fälts, kastas ett fel.
Fix förslag: Sätt url-fältet som optional i nested delta-representation ELLER dokumentera tydligt att hela store-statet (inkl. alla nested deltan) måste vara korrekt format.
Severity: Medium – agent-reader kunde inte publicera sitt fynd, men testet fortsatte med agent-security.
