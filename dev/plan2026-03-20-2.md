# AetherAgent – DDG Search Layer
**RFC · Nytt MCP-verktyg: `search`**  
**Datum:** 2026-03-20  
**Status:** Design klar – redo för implementation

---

## Problemet

AetherAgent är en perception-motor, inte en navigator. Den kan svara på
"vad finns på den här URL:en?" men inte "var finns svaret på den här frågan?".

```
Idag:
  "Hur många bor i Sverige?"  →  ??? ingen URL  →  AetherAgent kan inte börja

Med DDG-lagret:
  "Hur många bor i Sverige?"  →  search(q)  →  URL  →  fetch_stream_parse  →  svar
```

## Intern implementation

### Steg 1 – Bygg DDG URL

```rust
fn build_ddg_url(query: &str) -> String {
    let encoded = urlencoding::encode(query);
    format!(
        "https://html.duckduckgo.com/html/?q={}&kl=se-sv",
        encoded
    )
    // kl=se-sv  → svenska resultat
    // Ingen JavaScript krävs – Blitz Tier 1 räcker
}
```

### Steg 2 – fetch_stream_parse mot DDG

```rust
async fn search_ddg(query: &str, top_n: usize) -> Vec<SearchNode> {
    let url  = build_ddg_url(query);
    let goal = format!("hitta svar på: {}", query);

    // Återanvänder exakt samma pipeline som fetch_stream_parse
    let result = fetch_stream_parse(FetchStreamParseParams {
        url,
        goal,
        min_relevance: 0.25,   // lite lägre för sök-snippets
        top_n,
        max_nodes: 15,
    }).await?;

    result.nodes
}
```

### Steg 3 – Extrahera SearchResult från noder

DDG HTML-sidan har ett fast schema. Varje sökresultat är en `<div class="result">` med tre delar:

```
<h2 class="result__title">     → title + url
<a class="result__snippet">    → snippet-text
<span class="result__url">     → display-url
```

I det semantiska trädet mappar detta till:
- `role: "heading"` med länk → title + url
- `role: "cta"` eller `role: "text"` → snippet
- `role: "link"` med url-värde → klickbar URL

```rust
fn extract_results(nodes: Vec<SemanticNode>, top_n: usize) -> Vec<SearchResult> {
    let mut results = Vec::new();
    let mut current: Option<SearchResult> = None;

    for node in nodes {
        match node.role.as_str() {
            "heading" | "link" if node.value.contains("duckduckgo.com/l/?uddg=") => {
                // Extrahera riktig URL från DDG redirect
                let real_url = decode_ddg_redirect(&node.value);
                let domain   = extract_domain(&real_url);
                current = Some(SearchResult {
                    rank:       results.len() + 1,
                    title:      node.label.clone(),
                    url:        real_url,
                    domain,
                    snippet:    String::new(),
                    confidence: node.relevance,
                });
            }
            "cta" | "text" if current.is_some() => {
                if let Some(ref mut r) = current {
                    r.snippet = node.label.clone();
                    // Commit resultatet när vi har title + snippet
                    results.push(r.clone());
                    current = None;
                }
            }
            _ => {}
        }
        if results.len() >= top_n { break; }
    }

    results
}

fn decode_ddg_redirect(url: &str) -> String {
    // DDG-format: //duckduckgo.com/l/?uddg=ENCODED_URL&rut=...
    // Extrahera "uddg"-parametern och URL-decode den
    url.split("uddg=")
       .nth(1)
       .and_then(|s| s.split('&').next())
       .map(|s| urlencoding::decode(s).unwrap_or_default().to_string())
       .unwrap_or_default()
}
```

### Steg 4 – Direct answer detection

Om snippet-texten innehåller ett direkt svar (siffra, namn, datum) extraheras det:

```rust
fn detect_direct_answer(results: &[SearchResult]) -> Option<(String, f32)> {
    for r in results.iter().take(2) {
        // Enkla heuristiker för vanliga faktafrågor
        if let Some(num) = extract_number_answer(&r.snippet) {
            return Some((num, r.confidence));
        }
        if let Some(date) = extract_date_answer(&r.snippet) {
            return Some((date, r.confidence));
        }
    }
    None
}
```

## Pipeline – komplett flöde

```
Användarfråga: "Hur många bor i Sverige?"
        │
        ▼
┌──────────────────────────────────────┐
│  search(query, top_n=3)              │  ← NYTT VERKTYG
│                                      │
│  1. build_ddg_url(query)             │
│  2. fetch_stream_parse(ddg_url)      │  parse_ms: ~5ms
│  3. extract_results(nodes)           │
│  4. detect_direct_answer()           │
│                                      │
│  → SearchResult { results, answer }  │
└──────────────────────────────────────┘
        │
        ├─── direct_answer_confidence > 0.8?
        │         │
        │         YES → returnera direkt, KLAR
        │         │     "10 701 047 (källa: scb.se)"
        │         │
        │         NO  ▼
        │    ┌─────────────────────────────┐
        │    │  classify_request(url[0])   │  firewall-check
        │    │  fetch_stream_parse(url[0]) │  hämta källsida
        │    │  check_injection(answer)    │  trust-check
        │    │  → svar med källhänvisning  │
        │    └─────────────────────────────┘
        │
        ▼
    Returnera till LLM-agent
```

## Latens per scenario

| Scenariotyp | Steg | Estimerad tid |
|-------------|------|---------------|
| Direktsvar i snippet | DDG parse only | ~5–8ms |
| Faktasida behövs | DDG + källsida parse | ~15–40ms |
| JS-tung källsida | DDG + CDP Tier 2 | ~200–500ms |

## Trust Shield-integration

Alla svar passerar automatiskt genom befintlig trust-pipeline:

```rust
// Automatiskt i search()-svaret:
let injection_check = check_injection(&snippet_text).await?;
if !injection_check.safe {
    // Skippa detta resultat, ta nästa
    continue;
}

let firewall = classify_request(&url, &goal).await?;
if !firewall.allowed {
    // Blockerad domän, ta nästa
    continue;
}
```

Inga ändringar krävs i befintlig trust-kod.

## URL-rewriting (redan känt mönster)

Utöver DDG-redirect-decoding läggs dessa regler till:

```rust
fn normalize_url(url: &str) -> String {
    // DuckDuckGo standard → HTML-version (känt från session)
    if url.contains("duckduckgo.com") && !url.contains("html.duckduckgo.com") {
        return url.replace("duckduckgo.com", "html.duckduckgo.com/html/");
    }
    // Wikipedia → direkt (Blitz klarar det)
    // SCB → direkt (statisk HTML)
    url.to_string()
}
```

## Vad detta INTE löser (avsiktligt begränsat)

- **JS-tunga söksidor** – DDG HTML behöver ingen JS, så Blitz räcker alltid
- **Personalisering** – DDG HTML ger samma resultat för alla (bra för agenter)
- **Rate limiting** – DDG HTML har inga kända gränser för rimlig agenttrafik
- **Stöd för Tavily** – pluggbar arkitektur, byt backend utan att ändra `search()`-signaturen

---

## Nästa steg efter implementation

1. Lägg till `kl`-parametern för språkval (se-sv, en-us)
2. `df`-parameter för tidsfilter (past week, past month)
3. Pluggbar backend: `enum SearchBackend { DuckDuckGo, Tavily, SearXNG }`
4. Cache: samma query inom 60s returnerar cachat svar
