**Status:** Aktiv

## Sammanfattning

AetherAgent har nått Fas 16 – alla kärnfaser är klara. Projektet har gått från ren HTML-parser till en fullständig LLM-native browsermotor med 69 WASM-funktioner, 48 HTTP-endpoints, 32 MCP-tools.

**Fas 17: AetherDOM** är nästa stora milstolpe — en fullständig DOM-implementation med Arena-baserad allokering, Boa DOM Bridge, och hydration extraction.

---

## Nuläge — Kritiska begränsningar

```
HTML → html5ever → RcDom (Rc<Node>) → SemanticBuilder → SemanticTree → JSON → LLM
                        ↑                     ↑
                   heap-allokerat         Boa kan bara eval:a
                   ~1000 alloc/sida       rena uttryck (1+2)
                   cache-hostile          ingen DOM-access
```

| Problem | Nuvarande lösning | Konsekvens |
|---------|-------------------|------------|
| JS kan inte nå DOM | `js_eval.rs` blockar `document.*` | React/Vue/Angular-sidor = tomma skelett |
| Ingen querySelector | Regex-mönster i `js_bridge.rs` | Missar 90% av verkliga DOM-mönster |
| RcDom = långsamt | `markup5ever_rcdom` (Rc) | ~1000 heap-alloc/sida, cache-hostile |
| Ingen hydration | Ramverkdata ignoreras | SSR-sidor: kör JS i onödan |
| Ingen layout | Stubbar i `vision_backend.rs` | `getBoundingClientRect()` = nonsens |

### Vad som fungerar bra (ska INTE röras)

- SemanticBuilder pipeline (`parser.rs` → `semantic.rs` → `trust.rs`)
- Aho-Corasick injektionsdetektion
- Goal-driven streaming (95–99% token savings)
- Firewall + trust-system
- Vision pipeline (YOLOv8)
- MCP/HTTP/WASM API-yta

### Progressive Escalation Pipeline (Tier 0–4)

Nyckeln — köra minsta möjliga arbete per sida:

**Tier 0: Hydration Extraction (0 ms JS)**
- Regex/JSON-parser för `__NEXT_DATA__`, `__NUXT__`, `ng-state`, etc.
- 10+ ramverk: Next.js, Nuxt, Angular, Remix, Apollo, Gatsby, SvelteKit, Qwik, Astro, Next.js App Router (Flight Protocol)
- Om data extraheras → bygg SemanticTree direkt, SKIPPA JS
- **Uppskattning: täcker ~40% av moderna webbsidor**

**Tier 1: Statisk HTML Parse (nuvarande, ~1 ms)**
- `html5ever` → Arena DOM → SemanticBuilder
- Ingen JS-exekvering
- Fungerar för: statiska sidor, server-rendered utan hydration

**Tier 2: Sandboxed JS med DOM (NYTT, ~10–50 ms)**
- Boa + Arena DOM + `boa_class` bridges
- Kör `<script>`-block mot verklig DOM-representation
- Firewall-filtrering av fetch()/XHR (redan implementerat)
- MutationObserver-shim → fånga DOM-ändringar
- rAF-shim → köra animationsramverk
- Fungerar för: React SSR hydration, Vue client-side, Svelte

**Tier 3: Blitz Render (nuvarande, ~10–50 ms)**
- Ren Rust HTML/CSS rendering
- Layout-beräkning (riktig `getBoundingClientRect`!)
- Fungerar för: CSS-beroende layout, visuell grounding

**Tier 4: Chrome CDP (nuvarande, ~500–2000 ms)**
- Fullständig browser
- Allt stöds
- Fallback för: komplexa SPAs, WebGL, Canvas, Web Workers

**Intelligent escalation via TierHint:**
```rust
fn select_tier(html: &str, url: &str) -> Tier {
    // Tier 0: Hydration data finns?
    if let Some(data) = extract_hydration_state(html) {
        return Tier::Hydration(data);
    }
    // Tier 1: Ingen JS alls?
    let js = detect_js_snippets(html);
    if js.total_inline_scripts == 0 && !js.has_framework {
        return Tier::StaticParse;
    }
    // Tier 2: JS men inga tunga features?
    if !needs_layout(html) && !needs_canvas(html) {
        return Tier::BoaDOM;
    }
    // Tier 3: Behöver layout men inte full browser?
    if !needs_full_browser(html) {
        return Tier::BlitzRender;
    }
    // Tier 4: Allt annat
    Tier::ChromeCDP
}
```

### Hydration Extraction — 10+ ramverk utan JS

```rust
fn extract_hydration_state(html: &str) -> Option<HydrationData> {
    // Next.js Pages Router
    if let Some(data) = extract_next_data(html) {        // <script id="__NEXT_DATA__">
        return Some(HydrationData::NextJs(data));
    }
    // Next.js App Router (React Flight Protocol)
    if let Some(data) = extract_next_flight(html) {      // self.__next_f.push([...])
        return Some(HydrationData::NextFlight(data));
    }
    // Nuxt.js
    if let Some(data) = extract_nuxt_data(html) {        // window.__NUXT__=
        return Some(HydrationData::Nuxt(data));
    }
    // Angular Universal
    if let Some(data) = extract_angular_state(html) {    // <script id="ng-state">
        return Some(HydrationData::Angular(data));
    }
    // Remix
    if let Some(data) = extract_remix_context(html) {    // window.__remixContext
        return Some(HydrationData::Remix(data));
    }
    // ... Apollo, Gatsby, SvelteKit, Qwik, Astro
    None
}
```

**Varför detta är enormt:** ~40% av moderna produktionssidor använder SSR med hydration. Om vi kan extrahera datan direkt → 0 ms JS-exekvering, komplett data.

### Arena DOM — SlotMap ersätter RcDom

```rust
use slotmap::{SlotMap, new_key_type};

new_key_type! { pub struct NodeKey; }

#[derive(Trace, Finalize, JsData)]  // Boa GC-kompatibel
pub struct ArenaDom {
    nodes: SlotMap<NodeKey, DomNode>,
    document_key: NodeKey,
}

pub struct DomNode {
    pub node_type: NodeType,
    pub tag: Option<String>,
    pub attributes: Vec<(String, String)>,
    pub text: Option<String>,
    pub parent: Option<NodeKey>,
    pub children: Vec<NodeKey>,
    pub classes: Vec<String>,
    pub id: Option<String>,
}
```

| Egenskap | RcDom (nu) | SlotMap Arena |
|----------|-----------|---------------|
| Allokering | ~1000 heap-alloc/sida | 1 alloc (pre-allokerad Vec) |
| Cache | Hostile (Rc spridda) | Friendly (kontiguöst minne) |
| DFS-traversering | 1x (baseline) | 5–10x snabbare |
| Stale references | Möjlig (Rc cykler) | Omöjlig (generational index) |
| Boa-integration | Kräver wrapping | Direkt (#[derive(Trace)]) |
| WASM-storlek | Minimal | +50KB (slotmap crate) |
