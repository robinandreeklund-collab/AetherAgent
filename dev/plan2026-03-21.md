## Sammanfattning

AetherAgent har nått Fas 16 – alla kärnfaser är klara. Projektet har gått från ren HTML-parser till en fullständig LLM-native browsermotor med 69 WASM-funktioner, 48 HTTP-endpoints, 32 MCP-tools.

**Fas 17: AetherDOM** är nästa stora milstolpe — en fullständig DOM-implementation med Arena-baserad allokering, Boa DOM Bridge, och hydration extraction.

**Reviderad tidsuppskattning: 33–48 dagar** (ursprunglig plan sa 20–28 dagar).

---

## Kritiska insikter från djupanalys (2026-03-21)

Sex luckor identifierades och har integrerats i planen:

1. **Hydration-formaten är utdaterade** — Next.js App Router använder inte `__NEXT_DATA__` längre (React Flight Protocol istället), Nuxt/SvelteKit använder `devalue` (inte JSON), Qwik har "resumability" (inte hydration alls)
2. **Boa GC-problem** — `SlotMap` kan inte derivera `Trace` från `boa_gc` (extern typ). Lösning: **Rust äger arenan, JS får bara NodeKey-handles**
3. **`document.*` och `window.*` är hårdblockade** i sandboxen (FORBIDDEN-lista i `js_eval.rs:244-261`). Behöver **allowlist istället för blocklist**
4. **`parser.rs` har bara 2 tester, `semantic.rs` har 2** — man kan inte migrera med så lite skyddsnät. **Minst 30+ tester behövs FÖRE refactoring**
5. **Persistent Boa Context saknas** — varje `eval_js()` skapar ny `Context::default()`. DOM Bridge kräver **delad kontext med registrerade globaler**
6. **MutationObserver/rAF kräver event loop** — Boa har ingen. Uppskattningen 2–3 dagar → **5–7 dagar** (microtask-queue behövs)

## Nuläge — Kritiska begränsningar

```
HTML → html5ever → RcDom (Rc<Node>) → SemanticBuilder → SemanticTree → JSON → LLM
                        ↑                     ↑
                   heap-allokerat         Boa kan bara eval:a
                   ~1000 alloc/sida       rena uttryck (1+2)
                   cache-hostile          ingen DOM-access
```

| Problem | Nuvarande lösning | Konsekvens |
|---------|-------------------|------------|
| JS kan inte nå DOM | `js_eval.rs` blockar `document.*` | React/Vue/Angular-sidor = tomma skelett |
| Ingen querySelector | Regex-mönster i `js_bridge.rs` | Missar 90% av verkliga DOM-mönster |
| RcDom = långsamt | `markup5ever_rcdom` (Rc) | ~1000 heap-alloc/sida, cache-hostile |
| Ingen hydration | Ramverkdata ignoreras | SSR-sidor: kör JS i onödan |
| Ingen layout | Stubbar i `vision_backend.rs` | `getBoundingClientRect()` = nonsens |

### Vad som fungerar bra (ska INTE röras)

- SemanticBuilder pipeline (`parser.rs` → `semantic.rs` → `trust.rs`)
- Aho-Corasick injektionsdetektion
- Goal-driven streaming (95–99% token savings)
- Firewall + trust-system
- Vision pipeline (YOLOv8)
- MCP/HTTP/WASM API-yta

### Progressive Escalation Pipeline (Tier 0–4)

Nyckeln — köra minsta möjliga arbete per sida:

**Tier 0: Hydration Extraction (0 ms JS)**
- Extrahera SSR-data direkt från HTML utan att köra JS
- Om data extraheras → bygg SemanticTree direkt, SKIPPA JS
- **Uppskattning: täcker ~40% av moderna webbsidor**
- **OBS:** Ramverkens hydration-format skiljer sig kraftigt (se nedan):

| Ramverk | Datakälla | Format | Komplexitet |
|---------|-----------|--------|-------------|
| Next.js Pages Router | `<script id="__NEXT_DATA__">` | Standard JSON | Låg |
| Next.js App Router | `self.__next_f.push([...])` | React Flight Protocol (streamade chunks) | Hög |
| Nuxt 3/4 | `window.__NUXT__=` | **devalue** (inte JSON! — stöder Date, BigInt, Map, Set, cykler) | Medel |
| SvelteKit | `__sveltekit_*` namespace | **devalue** (samma som Nuxt) | Medel |
| Angular 17–19 | `<script id="ng-state">` + HTML-kommentarer (`ngh`) | JSON + incremental hydration annotations | Medel |
| Remix v2 / React Router v7 | `window.__remixContext` (möjligen `__reactRouterContext`) | JSON | Låg |
| Qwik | `<script type="qwik/json">` | **Resumability** (inte hydration!) — QRL-attribut, closure-serialisering | Hög |
| Astro | Inga inline-data — island-direktiv (`client:load`, `client:visible`) | HTML + direktiv | Låg |
| Apollo (GraphQL) | `window.__APOLLO_STATE__` | JSON | Låg |
| Gatsby | `window.___INITIAL_DATA___` | JSON | Låg |

**Kräver:** Custom `devalue`-deserializer (Nuxt+SvelteKit), React Flight Protocol-parser (Next.js App Router), QRL-parser (Qwik)

**Tier 1: Statisk HTML Parse (nuvarande, ~1 ms)**
- `html5ever` → Arena DOM → SemanticBuilder
- Ingen JS-exekvering
- Fungerar för: statiska sidor, server-rendered utan hydration

**Tier 2: Sandboxed JS med DOM (NYTT, ~10–50 ms)**
- Boa + Arena DOM + `boa_class` bridges
- Kör `<script>`-block mot verklig DOM-representation
- **Persistent Boa Context** — delad kontext per sida (inte per eval-anrop)
- **Allowlist-baserad säkerhet** — ersätter nuvarande blocklist i `js_eval.rs`
- Firewall-filtrering av fetch()/XHR (redan implementerat)
- Synkron DOM-diffing (iteration 1, enklare än full MutationObserver)
- rAF-shim med manuell frame-triggering
- Microtask-queue för `Promise.resolve().then()`
- **OBS:** Kräver event loop/microtask-queue — uppskattning 5–7 dagar (inte 2–3)
- Fungerar för: React SSR hydration, Vue client-side, Svelte

**Tier 3: Blitz Render (nuvarande, ~10–50 ms)**
- Ren Rust HTML/CSS rendering
- Layout-beräkning (riktig `getBoundingClientRect`!)
- Fungerar för: CSS-beroende layout, visuell grounding

**Tier 4: Chrome CDP (nuvarande, ~500–2000 ms)**
- Fullständig browser
- Allt stöds
- Fallback för: komplexa SPAs, WebGL, Canvas, Web Workers

**Intelligent escalation via TierHint:**
```rust
fn select_tier(html: &str, url: &str) -> Tier {
    // Tier 0: Hydration data finns?
    if let Some(data) = extract_hydration_state(html) {
        return Tier::Hydration(data);
    }
    // Tier 1: Ingen JS alls?
    let js = detect_js_snippets(html);
    if js.total_inline_scripts == 0 && !js.has_framework {
        return Tier::StaticParse;
    }
    // Tier 2: JS men inga tunga features?
    if !needs_layout(html) && !needs_canvas(html) {
        return Tier::BoaDOM;
    }
    // Tier 3: Behöver layout men inte full browser?
    if !needs_full_browser(html) {
        return Tier::BlitzRender;
    }
    // Tier 4: Allt annat
    Tier::ChromeCDP
}
```

### Hydration Extraction — Korrigerad för 2026

**OBS:** Formaten har ändrats kraftigt sedan 2024. Denna sektion är uppdaterad.

```rust
fn extract_hydration_state(html: &str) -> Option<HydrationData> {
    // === ENKLA (standard JSON) ===

    // Next.js Pages Router (legacy men vanligt)
    if let Some(data) = extract_next_data(html) {        // <script id="__NEXT_DATA__">
        return Some(HydrationData::NextJs(data));        // Standard JSON
    }
    // Angular Universal
    if let Some(data) = extract_angular_state(html) {    // <script id="ng-state"> + ngh-attrs
        return Some(HydrationData::Angular(data));       // Standard JSON + HTML-kommentarer
    }
    // Remix / React Router v7
    if let Some(data) = extract_remix_context(html) {    // window.__remixContext
        return Some(HydrationData::Remix(data));         // Standard JSON
    }
    // Apollo GraphQL
    if let Some(data) = extract_apollo_state(html) {     // window.__APOLLO_STATE__
        return Some(HydrationData::Apollo(data));        // Standard JSON
    }

    // === KRÄVER DEVALUE-DESERIALIZER ===

    // Nuxt 3/4 — OBS: INTE standard JSON!
    if let Some(data) = extract_nuxt_data(html) {        // window.__NUXT__=
        return Some(HydrationData::Nuxt(data));          // devalue-format (Date, BigInt, Map, Set, cykler)
    }
    // SvelteKit — OBS: INTE standard JSON!
    if let Some(data) = extract_sveltekit_data(html) {   // __sveltekit_* + teleported.set()
        return Some(HydrationData::SvelteKit(data));     // devalue-format
    }

    // === KRÄVER SPECIALPARSER ===

    // Next.js App Router — React Flight Protocol
    if let Some(data) = extract_next_flight(html) {      // self.__next_f.push([1, "..."])
        return Some(HydrationData::NextFlight(data));    // Streamade RSC-chunks, custom format
    }
    // Qwik — RESUMABILITY (inte hydration!)
    if let Some(data) = extract_qwik_state(html) {       // <script type="qwik/json">
        return Some(HydrationData::Qwik(data));          // QRL-attribut, closure-serialisering
    }

    // Astro — Inga inline-data, bara island-direktiv
    // Hanteras i Tier 1 (statisk parse) med client:* attributdetektering

    None
}
```

**Implementationsordning för hydration:**
1. **Vecka 1:** Enkla JSON-ramverk (Next Pages, Angular, Remix, Apollo) — snabba vinster
2. **Vecka 2:** `devalue`-deserializer (Nuxt 3 + SvelteKit) — kräver custom parser
3. **Vecka 3:** React Flight Protocol (Next App Router) — komplex streaming-format
4. **Vecka 4:** Qwik resumability — fundamentalt annorlunda modell

**Varför detta är enormt:** ~40% av moderna produktionssidor använder SSR med hydration. Om vi kan extrahera datan direkt → 0 ms JS-exekvering, komplett data.

### Arena DOM — DenseSlotMap ersätter RcDom

**Viktigt:** `SlotMap` kan INTE derivera `Trace` från `boa_gc` (extern typ). Lösning:
**Rust äger arenan, JS-sidan får bara `NodeKey`-handles.** Boa GC hanterar bara JS-wrapperobjekten.

```rust
use slotmap::{DenseSlotMap, new_key_type};

new_key_type! { pub struct NodeKey; }

// Rust-ägd — INTE exponerad till Boa GC
pub struct ArenaDom {
    nodes: DenseSlotMap<NodeKey, DomNode>,
    document_key: NodeKey,
}

pub struct DomNode {
    pub node_type: NodeType,
    pub tag: Option<String>,
    pub attributes: HashMap<String, String>,  // O(1) lookup (var Vec)
    pub text: Option<String>,
    pub parent: Option<NodeKey>,
    pub children: Vec<NodeKey>,
    pub classes: Vec<String>,
    pub id: Option<String>,
}

// JS-wrapper — exponerad till Boa via #[boa_class]
// Äger bara en NodeKey, inte själva datan
#[derive(Trace, Finalize, JsData)]
pub struct JsElement {
    key: NodeKey,  // Handle till Rust-ägd Arena
}
```

**Migrationsväg:** Option C (Hybrid) — behåll `html5ever` → `RcDom` för parsing, post-process till Arena via `rcdom_to_arena()`. Gradvis migration, ingen breaking change.

**SlotMap-variant:** `DenseSlotMap` (bäst för DOM-traversering, kontiguöst minne, snabbast iteration).

| Egenskap | RcDom (nu) | DenseSlotMap Arena |
|----------|-----------|---------------|
| Allokering | ~1000 heap-alloc/sida | 1 alloc (pre-allokerad Vec) |
| Cache | Hostile (Rc spridda) | Friendly (kontiguöst minne) |
| DFS-traversering | 1x (baseline) | 5–10x snabbare |
| Stale references | Möjlig (Rc cykler) | Omöjlig (generational index) |
| Attribut-lookup | O(n) linjär sökning | O(1) HashMap |
| Boa-integration | Kräver wrapping | NodeKey-handles (Rust äger data) |
| WASM-storlek | Minimal | +50KB (slotmap crate) |

**Alternativt crate:** `generational-arena-dom` — specifikt byggt för html5ever DOM med generational arenas. Utvärderas.
